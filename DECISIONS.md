# 技术决策记录

## ADR-0001：优先开发浏览器扩展而不是桌面应用

- 日期：2026-07-01
- 状态：已决定
- 背景：项目主要发生在 BOSS 直聘网页端，需要读取页面信息并辅助点击。
- 决策：MVP 使用 Chrome Extension Manifest V3。
- 理由：注入页面、读取 DOM、本地存储和用户控制面板都更直接。
- 影响：需要处理浏览器扩展权限、内容脚本隔离、第三方页面结构变化。

## ADR-0002：MVP 使用本地存储，不做云端账号系统

- 日期：2026-07-01
- 状态：已决定
- 背景：防重复记录和配置保存是 MVP 必需能力，但云端同步会引入账号、隐私和部署复杂度。
- 决策：MVP 使用 `chrome.storage.local` 保存配置和投递记录。
- 理由：简单、隐私风险低、适合单人使用和课程展示。
- 影响：换设备后数据不同步，后续可导入导出。

## ADR-0003：AI 回复先生成草稿，不自动发送

- 日期：2026-07-01
- 状态：已决定
- 背景：聊天回复涉及职业机会、薪资、隐私和语气，错误自动发送风险高。
- 决策：AI 只生成候选回复，由用户确认、修改、复制或发送。
- 理由：保留用户控制权，降低误发和不当回复风险。
- 影响：MVP 不追求全自动聊天闭环。

## ADR-0004：不实现绕过平台风控能力

- 日期：2026-07-01
- 状态：已决定
- 背景：平台可能存在验证码、限流、异常行为检测。
- 决策：项目不提供绕过验证码、隐藏自动化痕迹、突破投递限制等能力。
- 理由：保持学习和辅助工具边界，降低账号与合规风险。
- 影响：遇到验证码或限制时停止并提示用户手动处理。
