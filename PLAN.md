# 开发计划

## 阶段 0：项目规划

产出：

- `PROJECT.md`
- `PLAN.md`
- `TASKS.md`
- `BUGS.md`
- `DECISIONS.md`
- `CHANGELOG.md`

验收标准：

- 项目目标、MVP、风险边界、任务拆分清楚。
- 后续每个阶段都有明确可交付物。

## 阶段 1：浏览器扩展骨架

目标：

- 建立 Manifest V3 扩展基础结构。
- 支持扩展弹窗、内容脚本、本地存储。
- 能识别当前页面是否为 BOSS 职位列表页或聊天页。

建议技术栈：

- Vite
- React
- TypeScript
- Chrome Extension Manifest V3
- localStorage 或 chrome.storage.local

验收标准：

- 扩展能被 Chrome 加载。
- 弹窗能打开并展示基础状态。
- 内容脚本能注入目标页面。
- 非目标页面展示不可用状态。

## 阶段 2：配置与控制面板

目标：

- 用户可配置自我介绍、关键词、城市、HR 活跃时间、每日上限。
- 页面内控制面板展示开始、暂停、继续、停止、日志和统计。

验收标准：

- 刷新页面后配置仍保留。
- 控制按钮状态正确切换。
- 日志能显示最近操作。
- 空配置、错误配置有明确提示。

## 阶段 3：职位扫描与筛选

目标：

- 扫描列表页岗位卡片。
- 提取职位名、公司、地点、薪资、HR 活跃状态、详情链接或可点击节点。
- 根据条件输出候选、跳过及原因。

验收标准：

- 能稳定识别页面上已加载岗位。
- 关键词、城市、活跃时间筛选可独立验证。
- 每个跳过岗位有原因。
- 页面结构变化时不会导致扩展崩溃。

## 阶段 4：投递动作与防重复

目标：

- 对候选岗位执行“打开详情 -> 立即沟通 -> 发送话术”流程。
- 记录岗位 ID、HR、职位、公司、时间、结果。
- 支持暂停和任务恢复。

验收标准：

- 重复运行时已投递岗位会跳过。
- 达到每日上限后自动停止。
- 任一岗位失败不会影响后续岗位。
- 用户能导出或查看投递记录。

## 阶段 5：AI 智能回复草稿

目标：

- 用户上传或粘贴简历信息。
- 在聊天页识别最近 HR 问题。
- 生成候选回复草稿，用户确认后手动发送。

验收标准：

- AI 不直接自动发送。
- 草稿能结合简历信息和岗位上下文。
- 遇到薪资、隐私、敏感问题时给出谨慎建议。

## 阶段 6：测试、打磨与演示材料

目标：

- 完成手动测试清单。
- 准备 README、截图、演示流程、课程汇报材料。

验收标准：

- 核心流程有录屏或截图。
- 已知问题记录在 `BUGS.md`。
- `CHANGELOG.md` 能追溯每个阶段产出。

## 小步提交原则

- 每次只完成一个明确功能点。
- 每个阶段结束后更新 `TASKS.md` 和 `CHANGELOG.md`。
- 涉及页面选择器变更时同步记录在 `DECISIONS.md`。
